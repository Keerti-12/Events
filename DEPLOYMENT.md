# Deployment Guide: Lucky Event DJ

## Backend Deployment (Render)

### Step 1: Create Render Account
1. Go to [render.com](https://render.com) and sign up
2. Connect your GitHub account

### Step 2: Create PostgreSQL Database
1. Click **New** → **PostgreSQL**
2. Configure:
   - **Name**: `lucky-event-db`
   - **Database**: `luckyevent`
   - **User**: `luckyevent_user`
   - **Region**: Oregon (or closest to your users)
   - **Plan**: Free
3. Click **Create Database**
4. Wait for database to be created, then copy the **Internal Database URL**

### Step 3: Deploy Backend Web Service
1. Click **New** → **Web Service**
2. Connect your GitHub repository: `Keerti-12/Events`
3. Configure:
   - **Name**: `lucky-event-dj-api`
   - **Region**: Same as database
   - **Branch**: `main`
   - **Root Directory**: `backend`
   - **Runtime**: `Python 3`
   - **Build Command**: 
     ```
     pip install -r requirements.txt && python manage.py collectstatic --no-input && python manage.py migrate --no-input
     ```
   - **Start Command**: 
     ```
     gunicorn luckyevent.wsgi:application
     ```
   - **Instance Type**: Free

### Step 4: Configure Environment Variables (Render)

Add these environment variables in the Render dashboard:

| Key | Value | Notes |
|-----|-------|-------|
| `SECRET_KEY` | `<click Generate>` | Auto-generate a secure key |
| `DEBUG` | `False` | Production mode |
| `ALLOWED_HOSTS` | `.onrender.com` | Your Render domain |
| `DATABASE_URL` | `<from Step 2>` | Internal database URL from PostgreSQL |
| `CORS_ALLOWED_ORIGINS` | `https://YOUR-APP.vercel.app` | Your Vercel frontend URL (update after deploying frontend) |
| `SECURE_SSL_REDIRECT` | `True` | Force HTTPS |

**Optional Email Settings (for contact form notifications):**
| Key | Value |
|-----|-------|
| `EMAIL_HOST` | `smtp.gmail.com` |
| `EMAIL_PORT` | `587` |
| `EMAIL_HOST_USER` | `your-email@gmail.com` |
| `EMAIL_HOST_PASSWORD` | `your-app-password` |
| `DEFAULT_FROM_EMAIL` | `your-email@gmail.com` |

> **Gmail App Password**: Go to Google Account → Security → 2-Step Verification → App passwords → Generate

5. Click **Create Web Service**
6. Wait for deployment (2-5 minutes)
7. Note your backend URL: `https://lucky-event-dj-api.onrender.com`

### Step 5: Create Superuser (Admin Access)
After deployment, go to Render dashboard → Shell tab and run:
```bash
python manage.py createsuperuser
```
Follow prompts to create admin credentials.

## Frontend Deployment (Vercel)

### Step 1: Create Vercel Account
1. Go to [vercel.com](https://vercel.com) and sign up
2. Connect your GitHub account

### Step 2: Import Project
1. Click **Add New** → **Project**
2. Select your GitHub repository: `Keerti-12/Events`
3. Configure:
   - **Framework Preset**: Vite
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`

### Step 3: Configure Environment Variables (Vercel)

Add this environment variable:

| Key | Value |
|-----|-------|
| `VITE_API_URL` | `https://lucky-event-dj-api.onrender.com/api` |

> Replace with your actual Render backend URL from Step 7 above

4. Click **Deploy**
5. Wait for deployment (1-2 minutes)
6. Note your frontend URL: `https://YOUR-APP.vercel.app`

### Step 4: Update CORS on Render
Go back to Render dashboard and update:
- `CORS_ALLOWED_ORIGINS` → `https://YOUR-APP.vercel.app`

## Troubleshooting

### CORS Errors
Ensure `CORS_ALLOWED_ORIGINS` on Render matches your Vercel URL exactly (no trailing slash).

### 500 Errors
Check Render logs for Django errors. Common issues:
- Missing environment variables
- Database migration errors

### Static Files Not Loading
Verify `whitenoise` is in `INSTALLED_APPS` and middleware is configured.

### Database Connection Issues
Verify `DATABASE_URL` is using the Internal Database URL (not External).

## Local Development After Deployment

To run locally after deployment setup:

```bash
# Backend
cd backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver

# Frontend (new terminal)
cd frontend
npm install
npm run dev
```

Create a `.env` file in `backend/` for local development:
```env
SECRET_KEY=local-dev-secret-key-change-in-production
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1
CORS_ALLOWED_ORIGINS=http://localhost:3000,http://127.0.0.1:3000
```
